# 基于BiLSTM的语音情感识别

## 数据预处理
根据数据集在本地的存储路径生成音频列表文本文件，记录每条列表的路径
和情感标签，便于后续训练评估过程中读取音频。

观察 ESD 数据集中的音频样本，发现音频的长度基本在 1~3s 左右，对于训
练和评估的样本，为了避免显存溢出，我们对超过 3s 的音频进行了裁剪。对于
训练的样本，为了增强数据的多样性，我们进行了两种随机数据增强，包括语
速增强和音量增强。

## 特征提取
相较于传统的单一特征，我们选取了 Mel 倒谱系数、频谱质心、短时过零
率、均方根值能量等多种语音情感相关的音频特征，共计 312 维，更能反应语
音的情感特性。

## 模型训练
定义 BiLSTM 模型，对预处理后的音频数据进行逐个 epoch 迭代训练，这
里使用使用随机梯度下降（SGD）作为优化方法，并设置学习率、动量和权重
衰减。在每个 epoch 中，计算模型输出、损失和准确率，并进行反向传播和参
数更新。每个 epoch 结束后，使用测试集对模型进行评估，计算准确率。这里
保存相关数据到日志文件，可对模型训练的过程进行可视化。

## 模型评估
使用测试集对训练后的模型进行评估，获取分类报告并绘制混淆矩阵，比
较各参数下的模型性能，选取准确率最高的模型。

## 模型导出
读取训练模型，将训练过的模型加上一层 softmax 函数之后，导出为预测模
型，用于请按分类的预测。


# 文件介绍

## ESD文件夹

用于存放ESD数据集，如需重新训练模型，请自行下载，[下载地址](https://github.com/HLTSingapore/Emotional-Speech-Data)。

训练前请将ESD数据集中的1-20二十个文件夹放入ESD文件夹中

## configs文件夹

用于存放音频速度、音量两个数据增强器的配置信息

## *_utils文件夹

用于存放各种工具类和工具函数

## modules文件夹

定义模型

## train_perturbs文件夹

存放训练模型时使用的两种数据增强器

## sources文件夹

存放GUI的背景图以及和临时生成的图片，其中还放有一段示例音频，可以在运行GUI时使用

## output文件夹

存放输出的训练模型、训练日志、预测模型、以及混淆矩阵

# 运行说明

## create_data.py

创建标签列表、训练集列表、数据集列表、以及归一化特征文件

```shell
python -u create_data.py
```

（需要ESD数据集）

## train.py

训练模型

```shell
python -u train.py
```

（需要ESD数据集）

## evaluate.py

评估模型，绘制混淆矩阵

```shell
python -u evaluate.py
```

（需要ESD数据集）

## export_model.py

将训练模型导出为预测模型

```shell
python -u export_model.py
```

（output中有我们训练好的模型，可直接运行）

## app.py

运行音频情感文本识别应用

```shell
python -u app.py
```

（output中有我们导出的预测模型，可直接运行）
